# Quick Task 260609-lo4: Add Card modal defaults to Physical Card tab

## Goal

When the user clicks "Add Card", the modal opens on the **Physical Card** tab by
default. Previously it defaulted to the "From template" tab.

## Scope

Trivial state-default change in a single component — executed inline (no subagent
ceremony warranted for a 2-line literal change).

## Task

**File:** `src/components/cards/AddCardModal.tsx`

The modal's `mode` state (`"physical" | "template" | "custom"`) drives the active
tab. Two spots initialized it to `"template"`:

1. Initial `useState<Mode>("template")` → `useState<Mode>("physical")`
2. `resetForm()` `setMode("template")` → `setMode("physical")` — so reopening the
   modal also lands on Physical Card.

**Verify:** `npm run build` (tsc + vite) clean; `npm test` 18/18 green.
**Done:** Opening Add Card shows the Physical Card tab active by default.

## Human-verify

Open the app → Payments → "Add card". Confirm the **Physical Card** tab is active
on open, and after closing + reopening (resetForm path).
